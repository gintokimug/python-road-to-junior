    1) для привязки к объекту  необходимо к каждому аргументу прописывать self. 

    2)Ещё важно прописываать в названии функции __add__ __init__, то есть если ты в коде будешь использовать + - / * **  % тебе предварительно в функции(методе) нужно прописывть эту команду
    То есть: __add__ — это «кнопка», к которой физически припаян символ +. Если кнопки нет, символ не работает.
    Буквально: Символ + в Python — это просто «псевдоним» для метода __add__.
    Когда ты пишешь a + b, Python в ту же миллисекунду превращает это в a.__add__(b).
    Если ты назвал метод sum_salary, Python его не найдет, когда увидит +.
    Если ты назвал метод __add__, ты «подключился» к системной логике языка.


    3) метод super() ,с русского на русский понял, что он необходимо для копирования кода без повторного его ввода, `пример выше` показывает super() — это команда: «Игнорируй то, что написано в этой комнате, выйди за дверь и посмотри, как это делали старшие (в классе Manager) .Однако если нам нужно получить доступ к методу суперкласса из подкласса, мы используем функцию super().
    super().__init__(...) — база для любого сложного класса-наследника.

    4) `self.stock_options (без скобок) — это Атрибут (существительное). Это просто переменная, в которой лежит число. Ты просто «читаешь» её из памяти.`
    `super().get_total_salary() (со скобками) — это Метод (глагол). Это функция, внутри которой происходит какое-то вычисление (сложение базы и бонуса).`
    `Правило: Скобки () — это команда «Выполни действие!». Если ты их не поставишь, Python вернет тебе саму функцию (как описание процесса), а не результат её работы`

Так же существует несколько аксиом,догм хз. 
- **наследование** (5 видов)Одиночное наследование: дочерний класс наследует только от одного родительского класса.
- Множественное наследование: дочерний класс наследует от нескольких родительских классов.
- Многоуровневое наследование: дочерний класс наследует от родительского класса, который наследует от своего родительского класса.
- Иерархическое наследование: от одного родительского класса создается несколько дочерних классов.
- Гибридное наследование: сочетание нескольких форм наследования.
