# 1. **Python**(Python Lambda\Anonymous function)
- Python Lambda. Вызов анонимной функции для экономия памяти и выполнения необходимых циклов, объявляется lambda x : (выражение)
    Функции map(), filter() и reduce() — это «три кита», на которых строится обработка данных без использования явных циклов for. Их используют вместе с lambda, потому что это позволяет записывать сложные преобразования в одну строку, сохраняя код чистым (если не злоупотреблять).

    Почему их используют вместе с lambda?
lambda — это анонимная функция «на один раз». Передавать полноценную функцию через def в эти методы часто избыточно. Связка method(lambda, iterable) позволяет применить логику к каждому элементу коллекции мгновенно.
1. map() — Преобразование
Н   ужен, когда из одного списка нужно получить другой такой же длины, но с измененными значениями.
    Логика: Применить функцию к каждому элементу.

    Результат: Объект-итератор (нужно оборачивать в list())
*prices = [100, 500, 1000]*
*with_tax = list(map(lambda x: x * 1.2, prices))*
*[120.0, 600.0, 1200.0]*
2. Нужен, когда из списка нужно выкинуть лишнее по условию.

Логика: Функция внутри должна возвращать True или False. Если True — элемент остается.

Результат: Список меньше или равен исходному.

Пример: Оставить в списке только четные числа.

Python

*nums = [1, 2, 3, 4, 5, 6]*
*evens = list(filter(lambda x: x % 2 == 0, nums))*
*[2, 4, 6]*

3. reduce() — Свертка

Самый специфичный метод. Он «схлопывает» весь список в одно единственное значение.

Где взять: Нужно импортировать из библиотеки: from functools import reduce.

Логика: Берет два первых элемента, применяет функцию, результат берет как первый элемент и складывает со следующим.

Пример: Найти произведение всех чисел в списке.

Python

*from functools import reduce*
*nums = [1, 2, 3, 4]*
*product = reduce(lambda x, y: x * y, nums)*
*((1 * 2) * 3) * 4 = 24*

## 2. **Python**(Python Iterators)
    Итераторы — это методы, которые перебирают коллекции, такие как списки, кортежи и т. д. С помощью метода-итератора мы можем перебрать объект и вернуть его элементы.
    Технически объект-итератор в Python должен реализовывать два специальных метода, __iter__() и __next__(), которые вместе называются протоколом итератора.
    
    __iter__()возвращает сам объект итератора. При необходимости можно выполнить некоторую инициализацию.
    
    __next__()должен возвращать следующий элемент последовательности. При достижении конца последовательности и при последующих вызовах он должен вызывать исключение StopIteration.

class PowTwo:
    """Class to implement an iterator
    of powers of two"""

    def __init__(self, max=0):
        self.max = max

    def __iter__(self):
        self.n = 0
        return self

    def __next__(self):
        if self.n <= self.max:
            result = 2 ** self.n
            self.n += 1
            return result
        else:
            raise StopIteration


# create an object
numbers = PowTwo(3)

# create an iterable from the object
i = iter(numbers)

# Using next to get to the next iterator element
print(next(i)) # prints 1
print(next(i)) # prints 2
print(next(i)) # prints 4
print(next(i)) # prints 8
print(next(i)) # raises StopIteration exception

    Чтобы любой твой класс стал итератором, он должен следовать правилам «двух методов»:

__iter__(self): Этот метод должен вернуть сам объект итератора. Почти всегда тут просто пишут return self. Это нужно, чтобы Python понял: «Ок, этот объект можно запустить в цикл».

__next__(self): Это та самая «кнопка». Она должна:

Либо выдать следующее значение.

Либо, если данные закончились, прокричать: raise StopIteration.

Итератор — это объект-счетчик, который знает правило получения следующего элемента, но не хранит все элементы одновременно.
